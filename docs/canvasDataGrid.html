Class: canvasDataGrid

canvasDataGrid

canvasDataGrid(args)

Hierarchal canvas based data grid.
Parameters:
Name Type Description
args object Parameters for the grid.
Properties
Name Type Argument Default Description
parentNode object HTML element that will hold the grid.
name object <optional>
 Optional value that will allow the saving of column height and row width settings to the browser's local store. This name should be unique to this grid.
tree boolean <optional>
 false When true, an arrow will be drawn on each row that when clicked raises the expandtree event for that row and creates an inner grid.
treeHorizontalScroll boolean <optional>
 false When true, expanded child grids will scroll horizontally with the parent columns. When false, when scrolling horizontally child grids will remain stationary. This does not impact vertical scrolling behavior.
childGridAttributes object <optional>
 Attributes used for cell grids. These child grids are different than the tree grids. See the data property for more information about cell grids.
showNewRow boolean <optional>
 true When true, a row will appear at the bottom of the data set. schema[].defaultValue will define a default value for each cell. defaultValue can be a string or a function. When a function is used, the arguments header and index will be passed to the function. The value returned by the function will be the value in the new cell.
saveAppearance boolean <optional>
 true When true, and the attribute name is set, column and row sizes will be saved to the browser's localStore.
selectionFollowsActiveCell boolean <optional>
 false When true, moving the active cell with keystrokes will also change the selection.
multiLine boolean <optional>
 false When true, edit cells will be textareas, when false edit cells will be inputs.
globalRowResize boolean <optional>
 false When true, resizing a row will resize all rows to the same height. Questionable compatibility with hierarchical views.
editable boolean <optional>
 true When true cells can be edited. When false, grid is read only to the user.
allowColumnReordering boolean <optional>
 true When true columns can be reordered. NOT IMPLEMENTED.
showFilter boolean <optional>
 true When true, filter will be an option in the context menu.
pageUpDownOverlap number <optional>
 1 Amount of rows to overlap when pageup/pagedown is used.
persistantSelectionMode boolean <optional>
 false When true, selections will behave as if the command/control key is held down at all times.
rowSelectionMode boolean <optional>
 false When true, clicking on any cell will select the entire row that cell belongs to.
autoResizeColumns boolean <optional>
 false When true, all columns will be automatically resized to fit the data in them. Warning! Expensive for large (>100k ~2 seconds) datasets.
allowRowHeaderResize boolean <optional>
 true When true, the user can resize the width of the row headers.
allowColumnResize boolean <optional>
 true When true, the user can resize the width of the columns.
allowRowResize boolean <optional>
 true When true, the user can resize the row headers increasing the height of the row.
allowRowResizeFromCell boolean <optional>
 false When true, the user can resize the height of the row from the border of the cell.
allowColumnResizeFromCell boolean <optional>
 false When true, the user can resize the width of the column from the border of the cell.
showPerformance boolean <optional>
 false When true, the amount of time taken to draw the grid is shown.
debug boolean <optional>
 false When true, debug info will be drawn on top of the grid.
borderResizeZone number <optional>
 10 Number of pixels in total around a border that count as resize zones.
showHeaders boolean <optional>
 true When true, headers are shown.
showRowNumbers boolean <optional>
 true When true, row numbers are shown in the row headers.
showRowHeaders boolean <optional>
 true When true, row headers are shown.
schema boolean <optional>
 [] Sets the schema. See schema.
data boolean <optional>
 [] Sets the data. See data.
style canvasDataGrid.style <optional>
 {} Sets all style values overriding all defaults. Unless you're replacing 100% of the styles, don't use this property. See style property below.
Properties:
Name Type Description
selectedRows array Selected rows. Looks just like the data you passed in, but filtered for the rows the user has cells selected in. If any cell in the row is selected, all data for that row will appear in this array.
selectedCells array Jagged array of cells that the user has selected. Beware that because this is a jagged array, some indexes will be `null`. Besides the `null`s this data looks just like the data you passed in, but just the cells the user has selected. So if the user has selected cell 10 in a 10 column row, there will be 9 `null`s followed by the data from column 10.
changes array Array of changes and additions made to the grid since last time data was loaded. The data property will change with changes as well, but this is a convince list of all the changes in one spot. Calling `clearChangeLog` will clear this list.
input object Reference to the the edit cell when editing. Undefined when not editing.
controlInput object Input used for key controls on the grid. Any clicks on the grid will cause this input to be focused. This input is hidden behind the canvas.
currentCell cell Convenience object that represents the object that the mouse moved over last.
height number Height of the grid.
width number Width of the grid.
visibleCells array Array of cell drawn.
visibleRows array Array of visible row indexes.
selections array Matrix array of selected cells.
selectionBounds object object, bounds of current selection.
attributes object Object that contains the properties listed in the attributes section.
sizes object Mutable object that contains `sizes.columns` and `sizes.rows` arrays. These arrays control the sizes of the columns and rows. If there is not an entry for the row or column index it will fall back to the style default.
style canvasDatagrid.style Object that contains the properties listed in the style section. Changing a style will automatically call `draw`.
resizeMode string Represents the currently displayed resize cursor. Can be `ns-resize`, `ew-resize`, `pointer`, or `inherit`.
formatters canvasDatagrid.formatters Object that contains a list of formatting functions for displaying text. The properties in this object match the `schema[].type` property. For example, if the schema for a given column was of the type `date` the grid would look for a formatter called `formatters.date` if a formatter cannot be found for a given data type a warning will be logged and the string formatter will be used. Formatters must return a string value to be displayed in the cell.
sorters canvasDatagrid.sorters Object that contains a list of sorting functions for sorting columns.
filters canvasDatagrid.filters Object that contains a list of filters for filtering data. The properties in this object match the `schema[].type` property. For example, if the schema for a given column was of the type `date` the grid would look for a filter called `filters.date` if a filter cannot be found for a given data type a warning will be logged and the string/RegExp filter will be used.
data canvasDatagrid.data This is how data is set in the grid. Data must be an array of objects that conform to a schema. Data values can be any primitive type. However if a cell value is another data array, a child grid will be rendered into the cell. This child grid is different than a tree view grid and uses the `childGridAttributes` attribute for properties and styling.
schema canvasDatagrid.schema Schema is optional. Schema is an array of column objects. If no schema is provided one will be generated from the data, in that case all data will be assumed to be string data.
scrollHeight number The total number of pixels that can be scrolled down.
scrollWidth number The total number of pixels that can be scrolled to the left.
scrollTop number The number of pixels that have been scrolled down.
scrollLeft number The number of pixels that have been scrolled to the left.
offsetTop number The offset top of the grid.
offsetLeft number The offset left of the grid.
parentNode object The grid's parent HTML node.
isChildGrid boolean When true, this grid is a child grid of another grid.
openChildren boolean List of open child grids by internal unique row id.
childGrids boolean Child grids in this grid organized by internal unique row id.
height number Gets or sets the height of the grid.
width number Gets or sets the width of the grid.
parentGrid canvasDatagrid If this grid is a child grid, this is the grids parent.
canvas object The canvas element drawn onto for this grid.

Events

afterrendercell

Fires just after a cell is drawn. If you want to draw things in the cell, this is the event to attach to. Drawing on the canvas is allowed. Altering the context of the canvas is allowed.
Parameters:
Name Type Argument Description
ctx object Canvas context.
cell object <optional>
 Current cell.

beforebeginedit

Fires before a edit cell has been created giving you a chance to abort it. `e.preventDefault();` will abort the edit cell from being created.
Parameters:
Name Type Argument Description
cell object <optional>
 Cell object.

beforeendedit

Fires just before edit is complete giving you a chance to validate the input. `e.preventDefault();` will cause the edit to not end and row data will not be written back to the `data` array.
Parameters:
Name Type Argument Description
value object New value.
originalValue object Original value.
abort object Abort edit function. Call this function to abort the edit.
cell object <optional>
 Cell object.
textarea object Textarea or input HTMLElement depending on `attributes.multiLine`.

contextmenu

Fires when a context menu is requested. The menu item array can be altered to change what items appear on the menu. You can add items to the context menu but they must conform to canvasDataGrid.contextMenuItem. Removing all items from the list of menu items will cause the context menu to not appear. Calling `e.preventDefault();` will cause the context menu to not appear as well.

expandtree

Fires when the user clicks on the "drill in" arrow. When the arrow is clicked a new grid is created and nested inside of the row. The grid, the row data and row index are passed to the event listener. From here you can manipulate the inner grid. A grid is not disposed when the tree is collapsed, just hidden, but grids are not created until the arrow is clicked.
Parameters:
Name Type Description
grid object New, or if reopened existing, grid.
data object The row's data.
rowIndex object The row index that was expanded.

rendertext

Fires when text is drawn into a cell. If you want to change the color of the text, this is the event to attach to. To alter what text finally appears in the cell, change the value of `cell.formattedValue`. Keep in mind this text will still be subject to the ellipsis function that truncates text when the width is too long for the cell. You cannot alter the cell's height or width from this event, use `rendercell` event instead.
Parameters:
Name Type Argument Description
ctx object Canvas context.
cell object <optional>
 Current cell.
